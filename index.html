Projet : L'humour dans les modèles de langage

A. Énoncé du projet

Pourquoi, quand je demande à un modèle de langage comme ChatGPT, il n'est pas drôle. Si on demande à un modèle de créer une histoire fantastique, il sera bon, mais quand c'est une histoire triste, il créera une histoire avec un champ lexical lié à la mort et des choses de ce genre qu'il considère tristes. Cependant, il ne réussit pas à faire un texte qui nous fait réellement sentir tristes, contrairement à de bons écrivains qui nous font réellement ressentir les émotions. Dans ce projet, je vais me concentrer sur l'humour. Les LLM ne semble pas savoir ce qui est drôle. Mais au fait, qu'est-ce qui est drôle ? L'humour semble changer selon les cultures et est difficile à capturer. Dans le cadre de mon projet, je vais me pencher sur ce qu'est l'humour et pourquoi les LLM ont de la difficulté à en faire. Trouver une métrique pour évaluer l'humour d'un LLM. Ensuite, j'aimerais essayer différentes techniques pour rendre un LLM drôle. Par exemple, fine-tuner un modèle open source pour le rendre comique. En réalisent ce projet, je souhaite approfondir mes connaissances sur comment les LLM fonctionne.

B. Description détaillée

Spécification fonctionnelle :

  • Étudier et définir ce qu’est l’humour.
  • Élaborer une métrique pour évaluer l'humour produit par un LLM.
  • Analyser les performances actuelles des LLM pour la génération de contenu humoristique.
  • Implémenter des ajustements sur un modèle open source pour tester diverses approches d’optimisation de l’humour : fine-tuning, prompt engineering ou pré-entraînement sur des datasets humoristiques. Comparer les résultats avec la métrique définie.

Environnement et contraintes techniques :

  • Environnement de développement : Python, frameworks comme PyTorch avec des bibliothèques comme Hugging Face Transformers.
  • Modèle : Utilisation d’un modèle open source (probablement Llama).
  • Données : Création ou nettoyage d’un dataset humoristique.

C. Plan de développement

Durée totale : ~150 heures

Date de début : Dès validation du projet (~20 janvier).

Date de fin : Suivant le calendrier du cours (~25 avril).

Phases :

  1. Phase 1 : Définition projet (semaine 1-2)
    • Définition et planification du projet.
  2. Phase 2 : Recherche et définition (semaine 3-4)
    • Étude des théories de l’humour.
    • Identification des défis dans les LLM actuels.
  3. Phase 3 : Création d’une métrique et collecte des données (semaine 5-6)
    • Conception de la métrique d’évaluation.
    • Collecte et préparation d’un dataset humoristique.
    • Livraison : Dataset nettoyé, métrique opérationnelle (mi-février).
  4. Phase 4 : Développement et tests (semaine 7-11)
    • Implémentation du fine-tuning ou des autres techniques.
    • Tests et ajustements basés sur la métrique et des évaluations humaines.
    • Livraison : Résultats préliminaires des tests (mi-mars).
  5. Phase 5 : Analyse des résultats et finalisation (semaine 12-14)
    • Analyse des résultats
    • Réalisation rapport final.
    • Préparation de l'oral.
