Projet : L'humour dans les modèles de langage

A. Énoncé du projet

Explorer le monde de l'humour et de l'informatique puis essayer de créer un programme qui génère des pages 404 humoristique pour un site web.

B. Description détaillée

Spécification fonctionnelle :

  • Étudier l’humour dans l'informatique.
  • Élaborer une métrique pour évaluer l'humour produit par un LLM.
  • Analyser les performances actuelles des LLM pour la génération de contenu humoristique.
  • Implémenter des ajustements sur un modèle open source pour tester diverses approches d’optimisation de l’humour : fine-tuning, prompt engineering ou alignement. Comparer les résultats avec la métrique définie.

Environnement et contraintes techniques :

  • Environnement de développement : Python, frameworks comme PyTorch avec des bibliothèques comme Hugging Face Transformers.
  • Modèle : Utilisation d’un modèle open source (probablement Llama).
  • Données : Création ou nettoyage d’un dataset humoristique.

C. Plan de développement

Durée totale : ~150 heures

Date de début : ~20 janvier

Date de fin : ~25 avril

Avec le prof on va voir durant le projet donc se référer au rapports d'avancement.

D. Rapports d’avancement

    Semaine 1-2

    Inscription au cours et lecture de papier

    • With Great Humor Comes Great Developer Engagement
    • Long Live Software Easter Eggs
    • Exhaustive Survey of Rickrolling in Academic Literature
    • Generative AI to Generate Test Data Generators
    • Quantum humor is simultaneously funny and not.
    • 93% of Paint Splatters are Valid Perl Programs
    Semaine 3-4

    Expérimenter la création de page 404 avec un model en utilisant du prompt engineering. Comparer les models et les techniques.

    Semaine 5-6

    Demander à des personnes d'évaluer les réponse des différents models sur les différentes tâche. Composer un rapport sur les expérimentations. (Vous pouvez retrouver le rapport dans ce repo: https://github.com/GabrielBeaudoinUdem/ift4055)

    Semaine 7-8

    S'informer sur le fonctionnement des LLMs et rechercher sur les modèles non censurés et sur comment fine-tuner un LLM ou l'aligner.

    Semaine 9-10

    Créer un site web pour mener une expérimentation a plus grande échelle, générer un coprus de blague avec les 3 LLMs, les plus performant pour l'humour (Gpt-4o, deepseek-R1, gimini flash 2)

    Semaine 11-12

    Récolter les données de la grande étude et analyser les données.

    Semaine 13-14

    Écriture du rapport final et préparation de la présentation oral.
